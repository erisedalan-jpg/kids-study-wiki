## 红线（违反任何一条都视为生成失败）

1. **禁止编 URL、书名、ISBN、教材具体页码**。不确定一律写 🚧 或「待主会话核对」。
2. 「教材」行**只写占位**，由主会话回填 ChinaTextbook 路径。
3. 涉及具体数据/原理 → 末尾"生成校对"行已含「待家长核对」，但若你不确定数据，请在该数据后加 `（待家长核对）`。
4. 价值观题不下结论；避免性别 / 地域 / 民族刻板印象。
5. **若题材涉及古文、历史人物、政治文献、敏感议题（毒品、酗酒、性别认同、宗教争议等）**，**不要生成内容**，整篇只输出一行：

   ```
   __ROUTE_TO_OPUS__: <一句话说明为什么走主会话>
   ```

   下游脚本会据此把该词条转回主会话处理。

6. frontmatter `aliases` 数组**首位必须是 bare-name 原文**（与文件名一致），其后才是英文 / 同义词。

## 输出格式校验

- 整篇是合法 Markdown，frontmatter 用 `---` 包裹且为 YAML 合法
- 不输出任何前后缀（如 "好的，下面是..."、"```markdown" 包裹）
- 行末无多余空格；段间统一空一行
