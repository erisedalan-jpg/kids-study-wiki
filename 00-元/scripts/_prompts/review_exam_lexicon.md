## 检查清单（逐项判定 PASS / FAIL / WARN）

本模板是**知识密集型精简词条**，结构固定为：frontmatter → 一句话 → `## 🎓 考点精讲` → `## ⚠️ 高频易错点` → `## 🌐 中英对照（术语表）` → `## 📑 出处与参考资料` → `## 🔗 相关考点`。
**绝不要**因为「缺少绘本/家长话术/共读活动/中英例句/🧒📚 分层/单层#标题」而判 FAIL——这些区块在本模板中是**有意去除**的，去除是正确的。

1. **frontmatter 合规**：含 title / aliases / 学科 / 学段 / 状态 / 英文术语；`aliases` 数组首位 = bare-name 原文（与文件名去序号前缀一致）。无需「首次共读/最近共读」字段。
2. **学科锚定（最重要）**：`🎓 考点精讲` 是否用「{subject}」的真实知识（数学给定义/公式/定理；物理给物理量/公式/适用条件；化学给反应/方程式；生物给机制/过程；英语给语法规则/题型策略）。**若只有"结合起来思考""帮助理解"式泛化空话而无具体 {subject} 知识 → 判 FAIL，verdict=fix_major**。
3. **高考导向**：是否说明该考点在高考中的考查方式 / 解题关键 / 易错点；`⚠️ 高频易错点` 是否具体到 {subject} 知识层面（非空泛套话）。
4. **红线（不编 URL/书名/教材路径）**：`教材` 行应为「(待主会话核对教材索引)」占位；不确定数据应带「（待家长核对）」。
5. **术语表**：`🌐 中英对照（术语表）` 至少 3 行，列为「中文 | English | 说明」（**本模板不要求词性/例句列，缺这两列是正确的**）。
6. **相关考点**：`🔗 相关考点` 3–5 个 `[[bare-name]]`，应为**同学科**相邻考点；明显跨学科或编造无关词条 → WARN。
7. **敏感议题**：若词条名涉古文/历史人物/政治文献/敏感议题却生成了正文而非路由标记 → FAIL 并要求 reroute。

## verdict 判定标准

- `pass`：仅 0–1 个 WARN，无 FAIL
- `fix_minor`：有 WARN 或轻微 FAIL（如术语表 2 行、相关考点 2 个）
- `fix_major`：学科锚定失败（item 2 FAIL）/ 红线违规 / frontmatter 严重缺失
- `reroute_to_opus`：敏感议题误生成
