# 工作流：7 条触发短语

家长只需记这 7 条。每条都给具体例子。

## 1️⃣ 新增词条
> "新增词条：**[概念名]**，所属学科 **[学科]**，学段 **[学段]**"

例：`新增词条：彩虹，所属物理，学段 学前 小学`

→ Claude 检查重名 → 按模板生成完整内容（三层正文、中英对照、绘本、出处）→ 自动建立到相关词条的双向链接 → 给我审阅。

## 2️⃣ 把骨架填完整
> "把 **[词条名]** 从骨架升级到 **共读版完成 / 全龄完成**"

例：`把 红绿灯 升级到 共读版完成`

## 3️⃣ 批量骨架
> "为 **[学科]** 学段 **[学段]** 生成 **[主题]** 骨架，一批 10 篇"

例：`为 英语 学段 学前 生成 颜色和动物 骨架，一批 10 篇`

## 4️⃣ 共读回填
> "今天和孩子共读了 **[词条1] [词条2]**"

→ 更新 `最近共读` 字段；在 `杂物间/共读日记/YYYY-MM-DD.md` 写一条简记。

## 5️⃣ 找漏链
> "扫一遍 **[词条名 / 文件夹]**，看有没有该互链但没链上的"

## 6️⃣ 生成讲解 PPT（轻量版）
> "把 **[词条名]** 生成讲解 PPT（轻量）"

→ 调 `pptx` skill；纯文字 + 简单 emoji；输出 `素材/讲解PPT/[名]-轻量.pptx`

## 7️⃣ 生成讲解 PPT（精装版）
> "把 **[词条名]** 生成讲解 PPT（精装）"

→ 调 `nanobanana-ppt-skills`；含 AI 配图、设计感强；输出 `素材/讲解PPT/[名]-精装.pptx`

## 内容生成红线（每次新增/填充时遵守）

1. 涉及具体数据/原理 → 必须写"待家长核对"，并填 `📑 出处` 的"生成校对"行
2. 价值观题不下结论
3. 避免性别/地域刻板
4. 找不到来源就明说，不能编 URL 或书名
5. **新词条/升级词条**: `📑 出处` 的「教材」行**必须**引本地 ChinaTextbook 库, 路径查 [[00-元/教材索引]]. 学前/无对应版本时写 `(无对应教材, 待家长补充)`. 例:
   ```
   - **教材**: [[素材/教材/ChinaTextbook/小学/数学/人教版/义务教育教科书 · 数学一年级上册.pdf]] 第三单元
   ```
   规则只对 2026-05-06 之后新增/升级的词条生效, 已有词条不追溯改.
6. **学科目录新词条命名**: 按"首次出现学期"加 2 位序号前缀 (如 `16-加法.md`, 详见 [[00-元/命名规则#序号前缀（按学年顺序）]]). frontmatter `aliases` 数组**首位**必须含 bare-name (如 `aliases: [加法, addition, plus]`), 以保持现有 `[[016-加法|加法]]` 类链接通过 alias 解析仍然有效.

## 批量任务标准流程（≥ 30 词条时使用）

历经化学 520 / 生物 457 / 语文 571 等大规模批次，沉淀以下 5 步法。每一步对应 `00-元/scripts/` 下的可复用脚本——**不要再为每个学科一次性写脚本**。

### Step 1 · 冲突预检
计划写入的 bare-name 可能与已存在词条跨学科冲突（如 蛋白质 / 糖类 / DNA / 减数分裂 / 细胞核）。**先跑预检，再决定消歧后缀**：

```bash
python 00-元/scripts/check_naming_conflicts.py --candidates "蛋白质,糖类,DNA,减数分裂"
```

冲突项在创建前就分配后缀（`蛋白质（生物）.md`、`减数分裂（高中）.md`），避免事后改 frontmatter / aliases / 反链。

### Step 2 · 路由分配（主会话 vs 子代理 vs DeepSeek）
不是所有内容都适合 sonnet 子代理。按以下规则三路分流：

| 走主会话（Opus 4.7） | 走 sonnet 4.6 子代理 | 走 DeepSeek（脚本异步） |
|---|---|---|
| 古文 / 文言文（如离骚、赤壁赋） | 现代说明文（高质量） | 现代说明文（批量铺量） |
| 历史人物 / 政治文献（鲁迅、恩格斯、卢梭、林觉民） | 数理化生概念词条（中等复杂度） | 数理化生概念词条骨架（≥30 条批量） |
| 涉敏感话题（毒品、酗酒、性别） | 自然现象、动植物 | 真题题目文本清洗（pdfplumber 脏数据） |
| < 60 词条小批量 | 60–200 词条 | ≥ 200 词条大批量 / quota 受限时兜底 |
| 教材路径核对、终稿审校 | 跨学段串联、风格统一 | 中英对照表、轻量改写 |

**反例**：生物 457 词条全分给子代理，遭遇 daily quota（3am Asia/Shanghai 重置）+ weekly cap（5/15 重置）两次连击，最终主会话兜底 49 条。**对策（2026-05-12 起）**：批量铺量首选 DeepSeek 脚本，Sonnet/Opus 走复检；详见下方「DeepSeek 接入」。

### Step 3 · dispatch + monitor + retry
子代理批量分发用 `superpowers:dispatching-parallel-agents` skill。每批控制在 ~30 文件以内（避免单批超时），分批之间检查上批输出再决定是否继续。

```bash
# 批量分发后，跑 missing diff 找出失败项
python 00-元/scripts/check_missing.py 生物 expected_biology.txt > missing.txt
# 缺失数 < 50 直接主会话兜底；> 50 起新一批子代理
```

### Step 4 · 序号化
全部词条到位后用通用脚本统一序号化（**不要再写 renumber_xxx.py 一次性脚本**）：

```bash
python 00-元/scripts/renumber.py 化学              # 自动 3 位前缀
python 00-元/scripts/renumber.py 化学 --dry-run    # 先 dry-run 看一遍
```

排序键：frontmatter 学期（`_utils.XUEQI_ORDER`）→ 主题 → bare-name 字典序。

### Step 5 · 进度自动注入 CLAUDE.md
完成后跑统计脚本，幂等更新 CLAUDE.md `<!-- AUTO-PROGRESS-START/END -->` 锚点之间的进度块：

```bash
python 00-元/scripts/stats.py --write
```

### DeepSeek 接入（2026-05-12 启用）

针对 sonnet quota 易耗尽 + 大批量铺量成本敏感场景，引入 DeepSeek 官方 API 作为「生成主力」，Claude 系列退到「复检」角色。

**模型路由**（在 `_llm_router.py:MODEL_REGISTRY` 配置，运行前请校验 DeepSeek 官方文档 model 字段名）：

| Task | 模型 | 用途 |
|---|---|---|
| `SIMPLE` | `deepseek-v4-flash` | 短文本清洗、字段抽取、分类打标 |
| `COMPLEX` | `deepseek-v4-pro` | 词条骨架生成、长上下文改写 |
| `REASONING` | `deepseek-reasoner` (R1) | 多步推理、事实校验、字段一致性 |
| `REVIEW_OPUS` | `claude-opus-4-7` | 10% 抽检（最高质量） |
| `REVIEW_SONNET` | `claude-sonnet-4-6` | 40% 中量复审 |

**端到端流水线**：

```bash
# 0. 准备 .env（首次）
cp 00-元/scripts/.env.example 00-元/scripts/.env
# 填入 DEEPSEEK_API_KEY 和 ANTHROPIC_API_KEY
pip install -r 00-元/scripts/requirements-llm.txt

# 1. 生成阶段（DeepSeek 主力）
python 00-元/scripts/gen_atom_skeleton.py \
    --topics-file topics.jsonl \
    --model complex \
    --out-manifest 00-元/scripts/_llm_logs/政治-批次1.manifest.jsonl

# 2. 复检阶段（10% Opus / 40% Sonnet / 50% DeepSeek 自检）
python 00-元/scripts/review_dispatch.py \
    --manifest 00-元/scripts/_llm_logs/政治-批次1.manifest.jsonl

# 3. 红线复用既有脚本
python 00-元/scripts/check_naming_conflicts.py --candidates "..."
python 00-元/scripts/check_missing.py 政治 expected_政治.txt
```

**红线兼容**：
- DeepSeek 输出已在 prompt 中硬性要求「教材」字段只写占位 `(待主会话核对教材索引)`，由 Opus 兜底回填 ChinaTextbook 路径。
- 涉古文 / 敏感议题时模型返回 `__ROUTE_TO_OPUS__: <原因>`，`gen_atom_skeleton.py` 据此跳过并标记 `status: routed_to_opus`。
- 所有调用日志落 `00-元/scripts/_llm_logs/YYYY-MM-DD.jsonl`（已 gitignore），含 token / 延迟 / model，便于事后核算。

**反例对比**：
- 旧：生物 457 词条 → sonnet 子代理 → quota 双击 → 主会话兜底 49 条
- 新：1120 初中词条 → DeepSeek 生成（10/40/50 复检）→ Opus 只处理 `fix_major + reroute_to_opus`，预期 < 10% 流量

### content filter 应对
子代理被 content filter 过滤的词条不会报错——它们只是悄悄消失。Step 3 的 `check_missing.py` 是发现这类问题的唯一手段。常见触发：

- 古文整篇（先秦 / 唐宋八大家）
- 鲁迅、恩格斯、卢梭等历史/思想人物作品
- 毒品 / 酗酒 / 暴力等社会议题

→ 这些 Step 2 时直接走主会话，不要浪费子代理 quota。

## 脚本工具箱（`00-元/scripts/`）

| 脚本 | 用途 |
|---|---|
| `_utils.py` | 共享：UTF-8 setup / 学期映射 / 学科目录清单 / frontmatter 解析 |
| `renumber.py` | 学科目录词条序号化（参数化，替代一次性脚本） |
| `stats.py` | 统计词条数 → 注入 CLAUDE.md 进度块 |
| `check_naming_conflicts.py` | 跨学科 bare-name 重名预检 |
| `check_missing.py` | expected vs actual diff，子代理失败后用 |

所有脚本都 `from _utils import setup_utf8` 解决 PowerShell GBK 乱码。日志统一英文（`OK:` / `SKIP:` / `DRY:`），中文路径仅作为数据传递。
