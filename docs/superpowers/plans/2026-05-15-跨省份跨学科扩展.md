## 1. 通用执行工具与流程模板

### 1.1 单张卷"端到端"标准化命令序列

每张卷处理 5 步（1-2 自动；3 是 sonnet 子代理；4-5 自动）：

```bash
# Step 1: PDF → 题块 JSON
python "00-元/scripts/parse_exam_pdf.py" \
  --pdf "<PDF 绝对路径>" \
  --subject <学科> --province <省份> --year <年份>

# Step 2: 候选 tag（白名单关键词命中）
python "00-元/scripts/tag_questions.py" \
  --questions "docs/superpowers/working/<省份>-<学科>-<paper_id>-questions.json" \
  --lexicon "00-元/scripts/_<学科>_lexicon.json"

# Step 3: LLM 终审（dispatch sonnet 子代理；prompt 见 1.2）
# 子代理读 questions JSON + 候选 tag，输出 tags/gap_terms/difficulty 字段，原地写回

# Step 4: 渲染原子词条
python "00-元/scripts/render_exam_atoms.py" \
  --questions "docs/superpowers/working/<省份>-<学科>-<paper_id>-questions.json"

# Step 5: 重跑聚合（建议每个 phase 完成后跑一次，不必每张卷跑）
python "00-元/scripts/aggregate_exam_indices.py" --province <省份> --subject <学科>
```

### 1.2 sonnet 子代理 LLM 终审 Prompt 模板

```
你是高考<学科>考点标注员。我会给你一张<年份>年<省份><学科>卷的题块 JSON，每题已带 tag_candidates（白名单关键词命中候选）。

任务：对每题修改 JSON 加三个字段（不改原字段）：
1. tags: 从 tag_candidates 中选 2-4 个最贴合该题考点的 term。必须从候选中选，不能创造新词。如果候选为空或都不贴合，留 [] 并依靠 gap_terms。
2. gap_terms: 题面里出现重要<学科>概念但不在候选里（白名单缺口），最多 3 个。
3. difficulty: 易/中/难（启发式：选择 Q1-6/填空 Q12-13=易；解答倒 2 题/多选最后一题=难；其他=中）。

输入 JSON 路径：<absolute path>
输出：原地写回 JSON（保留所有原字段，加新字段）。完成后报告：N 题完成 / X 题待人工核对（tags 为空）。
```

```python
def _classify_qtype(stem: str, qno: int, total_q: int, subject: str = "数学") -> str:
    """题型分类（按学科 + 题号位置）。"""
    if subject == "数学":
        # 现有逻辑
        ...
    elif subject in ("物理", "化学", "生物"):
        # 物化生卷典型结构（差异不大）：
        # 选择题 1-7（理综单科版）或 1-13（单科卷）
        # 实验/解答 8-12（理综）或 14+（单科）
        if total_q <= 14:

            return "选择" if qno <= 7 else "解答"

            return "选择" if qno <= 13 else "解答"
    return "解答"
```

## Phase D: 3省 语文/英语（大改造）

### D.1 前置：流水线大改

语文/英语真题结构与数学完全不同：
- **语文**: 现代文阅读（多选+主观）+ 古诗鉴赏（主观）+ 文言文翻译 + 默写 + 作文
- **英语**: 听力（听录音回答）+ 完形填空 + 阅读理解 + 七选五 + 短文改错 + 作文

需要新写：
1. `parse_exam_pdf.py` 加学科分支：按"现代文阅读"/"古代诗文阅读"等大题标题切分而非按"1./2./..." 切分
2. `_classify_qtype` 重写：语文区分"概念题/默写题/翻译题/鉴赏题/作文题"等
3. LLM 终审 prompt 加学科上下文：作文题的"考点"应为写作要素（议论文/记叙文）+ 主题（家国情怀/科技人文）
4. 词条 frontmatter 可能要加新字段（如"作文主题"/"文言文出处"等）

### D.2 改造工作量

约 **2-3 天**重写 + 验证。建议作为独立 phase。

**暂缓建议**：先把 Phase A/B/C 跑完（数学+物化生 6 学科 × 3 省 = 18 个卷组），再单独评估语文/英语是否值得做。理由：
- 数学+物化生学科考点离散化容易，价值清晰
- 语文/英语考点连续化（写作要素），打 tag 价值低，更适合"经典真题反复研读"而非"高频考点统计"

### D.3 若决定推进，顺序

1. 先做 1 张语文 Pilot（如 2024 新课标Ⅱ 语文）验证 split 逻辑
2. 通过后再批量
3. 英语跟语文同样模式

## 5. 风险与回退

| 风险 | 触发条件 | 回退 |
|---|---|---|
| sonnet quota 5/15 后再次紧张 | 单日 token 上限触发 | 分多日跑，或退回主会话 LLM 终审（精度不变，速度慢 4-6 倍） |
| 北京数学 PDF 切分失败 | parse 输出题数明显异常 | mini-Pilot 时发现，加 paper_aliases 北京特例规则 |
| 物化生 PDF 公式失真严重 | 题面 OCR 错误率 > 30% | 等用户人工核对 / 暂停该学科 / 引入 mathpix |
| 理综卷拆分困难 | 3 学科混编无法清晰归科 | 标记 paper="理综"，按学科 lexicon 过滤即可 |
| 跨省真题 bare-name 撞车 | analyze_links 显示重复 bare-name | 已知设计副作用，不影响反链文本；后续可在 render 时把"省份"加进 bare-name |
