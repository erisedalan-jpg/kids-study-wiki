## 1. 目录结构

物理文件夹**只按学科扁平分**，不区分学段。学段、主题、状态等元信息全部放进每篇文章的 frontmatter，靠 Dataview 自动生成索引。

```
study/
├── 00-元/
│   ├── 模板/
│   │   └── 词条模板.md
│   ├── 命名规则.md
│   └── 工作流.md
├── 数学/             # 全学段（学前→高中）
├── 英语/             # 全学段
├── 语文/             # 占位
├── 物理/             # 占位（小学"科学"中物理类内容也归这）
├── 化学/             # 占位
├── 生物/             # 占位
├── 历史/             # 占位
├── 地理/             # 占位
├── 政治/             # 占位
├── 生活与社会/       # 跨学科：红绿灯、超市、医生、节日等
├── 索引/             # Dataview 视图（查询代码块手写一次，结果自动生成）
├── 杂物间/           # 共读日记、孩子的问题、临时笔记
│   └── 共读日记/
└── 素材/             # 本地资源附件
    ├── 教材/
    ├── 绘本/
    ├── 图片/
    ├── 音视频/
    └── 讲解PPT/      # 第 6 节生成的 .pptx 输出位置
```

### 命名规则

- 文件名 = 中文概念名（例：`光.md`、`凑十法.md`、`红绿灯.md`）
- 英文别名挂在 frontmatter 的 `aliases` 字段，`[[light]]` 也能跳到《光》
- 跨学段概念（数数、光、彩虹）**只放一份**，归到它的"主学科"文件夹，靠 frontmatter `学段: [学前, 小学, 初中]` 让它在多个学段索引中出现
- 小学"科学"课内容按主题拆到对应学科（光→物理、植物→生物、岩石→地理）

### 跨学科概念归属规则

- 同时涉及多学科的概念（例如"光"属物理也属诗词意象），**主学科归位 + 在其它学科建短链词条**
- 短链词条只含 frontmatter + 一句话 + `[[主词条]]` 跳转，不重复正文

## 2. 词条模板

每篇文章使用统一模板，区块固定，便于 LLM 生成一致、便于阅读检索。模板存放于 `00-元/模板/词条模板.md`。

````markdown
---
title: <概念名>
aliases: [<英文别名>, <其他叫法>]
学科: <数学/英语/物理/...>
学段: [学前, 小学, 初中, 高中]   # 实际覆盖的所有学段
主题: [<子主题1>, <子主题2>]
状态: 骨架                         # 骨架 / 共读版完成 / 全龄完成
英文术语: <english term>
首次共读:
最近共读:
---

# <概念名>

> **一句话**：<10-25 字的核心定义>
> **English**: <一句话英文版>

---

## 🎓 给 12+（进阶版）

涉及科学原理、公式、定理；链接到正式课程内容。
````

### 模板设计要点

| 区块 | 解决的问题 |
|---|---|
| frontmatter `状态` | 进度可见；Dataview 可自动列出"今天能讲的" |
| 三层正文 | 单页分层（方案 C 核心），孩子长大顺一页往下读 |
| 中英对照（词级 + 句级） | 词查得快、句可朗读 |
| 相关绘本 | 共读时直接对接实体书 |
| 家长讲解话术 | wiki 的杀手区——打开就能讲，不用二次翻译 |
| 📺 讲解版 | 可直接用 Obsidian Slides 核心插件全屏对孩子开讲；也作为 PPT 生成的源 |
| 出处与参考资料 | LLM 内容可溯源；"生成校对"行让事实可追责 |
| 双向链接 | Karpathy wiki 精神的具体实现 |

### 不做的取舍（YAGNI）

- ❌ 不要"修订历史"区块（git 已记录）
- ❌ 学前阶段不加拼音（孩子不是直接读者）；将来需要时再以插件方式处理
- ❌ 不要在每页放音频朗读链接（YAGNI；将来如有需要再批量加）

## 3. 骨架生成策略

### 核心原则：**先建图，后填血肉**

骨架 = 只填 frontmatter + 一句话定义 + 相关词条链接 + 各正文区块的 🚧 占位符。**不填三层正文**。

这样做的好处：
- 一次性几十篇骨架 2-3 小时搞定
- 任何文章里点 `[[彩虹]]` 不变红链
- 真正写正文按需触发，避免内容堆积失控

### 骨架文件示例

```markdown
---
title: 彩虹
aliases: [rainbow]
学科: 物理
学段: [学前, 小学]
主题: [自然现象, 光学]
状态: 骨架
英文术语: rainbow
---

# 彩虹

> **一句话**：雨后阳光穿过小水珠，分出七种颜色。
> **English**: A rainbow forms when sunlight passes through water droplets.

🚧 详细内容待写。

## 🔗 相关
[[光]] · [[颜色]] · [[雨]] · [[太阳]]
```

### 第一批骨架范围

按当前最迫切需求（5 岁 / 3 岁孩子 + 数学/英语/生活与社会）：

| 学科文件夹 | 骨架数 | 内容范围 |
|---|---|---|
| `数学/` | ~20 | 学前数学启蒙：数数、1-10、形状、大小、长短、时间、钱、加法、减法、对称…… |
| `英语/` | ~15 | 学前主题词汇：颜色、动物、家庭、身体、食物、天气、数字、问候、动作词…… |
| `生活与社会/` | ~25 | 高频生活场景：红绿灯、超市、医院、银行、公交、医生、警察、消防员、节日（春节/中秋/…）、刷牙、生病、迷路怎么办…… |
| **总计** | **~60** | 覆盖学前→小学一年级日常 |

> ⚠️ K-12 学业骨架先不做。等孩子真到那个学段再生成对应学科——一次只做眼前 1-2 年，否则会被一千个红链劝退。

### 内容生成红线

约束 LLM 生成时必须遵守：

1. **科学事实**：涉及具体数据/原理（光速、行星距离、动物分类）必须明确标"待家长核对"，并填 `📑 出处` 区块的"生成校对"行
2. **价值观/品格**：用陈述不下结论。"为什么不能打人"——给规则和示例，不写"打人的孩子是坏孩子"
3. **避免性别刻板**：医生不默认男、护士不默认女
4. **避免地域刻板**：节日尽量含中外多元
5. **不编造来源**：找不到权威来源就明说，不能编 URL 或书名

## 4. 日常工作流：7 条触发短语

家长只需记住这 7 条，剩余交给 Claude Code。

### 1️⃣ 新增词条（最常用）
> "新增词条：**[概念名]**，所属学科 **[数学/英语/...]**，学段 **[学前/小学/...]**"

例：`新增词条：彩虹，所属物理，学段 学前 小学`

→ Claude 检查是否已存在 → 按模板生成完整内容（三层正文、中英对照、绘本、出处） → 自动建立到相关词条的双向链接 → 给家长审阅。

### 2️⃣ 把骨架填完整
> "把 **[词条名]** 从骨架升级到 **共读版完成 / 全龄完成**"

→ Claude 读现有 frontmatter 与占位 → 补齐对应正文区块 → 更新 `状态` 字段。

### 3️⃣ 批量骨架
> "为 **[学科]** 学段 **[学前/小学/...]** 生成 **[主题]** 骨架，一批 10 篇"

例：`为 英语 学段 学前 生成 颜色和动物 骨架，一批 10 篇`

→ Claude 一次给 10 篇，等家长点头再要下一批。

### 4️⃣ 共读回填
> "今天和孩子共读了 **[词条1] [词条2]**"

→ Claude 更新这些词条的 `最近共读` 字段；在 `杂物间/共读日记/YYYY-MM-DD.md` 写一条简记。

### 5️⃣ 找漏链
> "扫一遍 **[词条名]** / **[文件夹]**，看有没有该互链但没链上的词条"

→ Claude 用 grep 找概念名出现处 → 报候选列表给家长确认 → 点头后批量加 `[[wiki-link]]`。

### 6️⃣ 生成讲解 PPT（轻量版）
> "把 **[词条名]** 生成讲解 PPT（轻量）"

→ Claude 调用 **`pptx` skill**，基于词条「📺 讲解版」区块（或自动从三层正文抽取）生成纯文字 + 简单 emoji/图标的 .pptx → 输出到 `素材/讲解PPT/[词条名]-轻量.pptx`

**适用**：日常临时讲解、追求快/稳/可重复。

### 7️⃣ 生成讲解 PPT（精装版）
> "把 **[词条名]** 生成讲解 PPT（精装）"

→ Claude 调用 **`nanobanana-ppt-skills`**，含 AI 配图、设计感更强的版式 → 输出到 `素材/讲解PPT/[词条名]-精装.pptx`

**适用**：周末备课、给爷爷奶奶发的、想拍照发朋友圈的。

## 5. Obsidian 推荐插件

### 🔴 必装

- **Dataview** —— 用 SQL 风格查询自动生成索引页。frontmatter 不装这个就是死数据。
- **Slides（核心插件，非社区）** —— Settings → Core plugins → Slides 开启。让「📺 讲解版」区块按 `---` 分页全屏播放。

### 🟡 强烈推荐

- **Templater** —— 新建笔记自动套词条模板填好 frontmatter，配快捷键一键开新词条
- **Obsidian Git** —— 自动 commit + 推送到 GitHub 私有仓库。**避免几年内容丢失**
- **Various Complements** —— 写 `[[` 时弹已有词条建议，少打错字、少建重复

### 🟢 可选

- **Excalidraw** —— 词条里手绘草图（讲"光的折射"时一笔画三棱镜）
- **Image Toolkit** —— 点图放大（适合给孩子看绘本插图）
- **Calendar** —— 共读日记按日历显示
- **Mind Map** —— 把双链网络可视化（孩子升小学后好用）

### 📵 不建议装

- 任何 AI 写作插件（已有 Claude Code，冗余）
- 任何复杂看板/项目管理插件（YAGNI）

### 开箱即用的 Dataview 视图

放在 `索引/` 下：

`索引/状态-未完成.md`：

````markdown
# 状态-未完成

## 骨架待写
```dataview
TABLE 学科 as 学科, 学段 as 学段
FROM ""
WHERE 状态 = "骨架"
SORT file.mtime DESC
```
````

`索引/按学段-学前.md`：

````markdown
# 学前可读

```dataview
TABLE 学科 as 学科, 状态 as 状态
FROM ""
WHERE contains(学段, "学前") AND 状态 != "骨架"
SORT 学科, file.name
```
````

类似可建：`按学段-小学.md`、`按学段-初中.md`、`按学段-高中.md`、`最近共读.md`、`待家长核对.md`。

## 6. 实施路线图

阶段化推进，每阶段独立可用，不必等下一阶段才能见效。

### 阶段 0：奠基（30 分钟，今天就能做）
- 创建文件夹结构（10 个学科 + 元 + 索引 + 素材）
- 写 `00-元/模板/词条模板.md`
- 写 `00-元/命名规则.md`、`00-元/工作流.md`
- 创建 `索引/` 下 4-5 个 Dataview 视图
- Obsidian 启用 Slides 核心插件，安装 Dataview + Templater + Obsidian Git
- 配置 Templater 让"新建笔记"默认套词条模板

### 阶段 1：骨架批次 1（1-2 小时）
- 用触发短语 3️⃣ 批量生成 60 篇学前骨架
- 三个学科，每批 10 篇，分 6 批，每批扫一遍

### 阶段 2：试用 1 周
- 不主动新增内容，只在孩子真问到时用 1️⃣ 触发，把对应骨架填完整或新建词条
- 每周日做一次回顾：用触发 5️⃣ 找漏链、用 4️⃣ 回填共读记录
- 评估：模板/触发短语是否需要微调

### 阶段 3：扩展（按月）
- 第 1 个月：补完 60 篇学前骨架的"共读版正文"
- 第 2 个月：试出第一批"📺 讲解版"+ PPT 生成（每月做 4-5 个精装版）
- 第 N 月：根据孩子升学进度，用 3️⃣ 扩到下个学段

### 阶段 4：长期维护（持续）
- 每季度审一次"📑 出处与参考资料"是否陈旧
- 孩子识字后开放给孩子自己浏览，观察哪些词条孩子真在读，作为内容优先级信号

## 7. 风险与缓解

| 风险 | 缓解策略 |
|---|---|
| 内容堆积失控、骨架变红链海 | 严格执行"按需填充"——不预先写正文 |
| LLM 事实错误流入 wiki | 每篇必填"生成校对"行；季度审核；避免对孩子讲未核对内容 |
| 多年积累内容丢失 | Obsidian Git 自动同步到私有 GitHub 仓库 |
| 孩子识字后发现旧内容写法低龄、不愿意读 | 单页分层结构本就支持往深处读；最坏情况按 1️⃣ 重写 |
| 工作流过于繁琐导致家长放弃 | 7 条触发短语已是 MVP；阶段 2 试用后据反馈再压缩 |
| 不同学段同一概念归属冲突 | 主学科归位 + 其它学科建短链；frontmatter `学段` 多选 |

---

## 8. 不做的事（YAGNI 边界）

- ❌ 不做语音朗读自动生成（如需要，孩子识字时再加）
- ❌ 不做习题/测验系统（这是 wiki，不是 LMS）
- ❌ 不做学习进度跟踪/打卡（家长心智负担）
- ❌ 不做多人协作/账号系统（单家庭使用）
- ❌ 不预先生成 K-12 全部骨架（按学段推进）
- ❌ 不做 Obsidian 之外的前端（Web/App）—— 长期保持 markdown 单一来源
- ❌ 不做自动事实核查 agent（LLM 自己核 LLM 不可靠；保留人工核对）
