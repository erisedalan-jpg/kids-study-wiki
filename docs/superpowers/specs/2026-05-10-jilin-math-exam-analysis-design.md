# 吉林数学高考真题分析 / 真题分析流水线设计稿

## 0. 背景

`素材/真题/` 下新增了 北京 / 吉林 / 黑龙江 三省 2008-2024 高考真题（9 学科）。本次先以**吉林省数学**为切入点，跑通 **PDF → 题目原子化 → 考点 tag → 知识库反链 → 多维度分析索引** 的端到端流水线。

试点目标：
1. 不止产出"吉林数学分析报告"，更要沉淀一套**省份/学科参数化**的可复用脚本。
2. 用真题反向校准与扩充现有 526 个数学词条（高频考点排序 + 缺口词条清单 + 例题反链）。
3. 验收通过后，可平移到北京/黑龙江、扩展到英语/物理/化学等其他学科。

## 1. 范围决策

| 决策点 | 选择 | 备注 |
|---|---|---|
| 试点覆盖 | 2015-2024 吉林数学，文+理 ≈ **19 套** | 2023-2024 不分文理；约 420 道题 |
| 知识库形态 | **题库 + 索引** 双层 | 真题原子化为词条 + 索引层做统计 |
| 分析方向 | ① 高频考点统计 ② 反链回填 ③ 缺口词条挖掘 ④ 高频考点→代表例题 ⑤ 题型×考点交叉表 | 文理差异对比此次不做 |

## 2. 目录与命名

新增（不动现有 7 学科目录）：
```
真题/<省份>-<学科>/                  # 试点：真题/吉林-数学/
  YYYY-<文/理/卷别简称>-NN.md        # 一道题 = 一个词条
索引/真题/                          # 新建子目录
  <省份><学科>-高频考点.md           # 主交付物 ①
  <省份><学科>-题型×考点交叉表.md    # 主交付物 ⑤
  <省份><学科>-缺口词条清单.md       # 主交付物 ③
  <省份><学科>-试卷地图.md           # 副产品
00-元/scripts/
  exam_pipeline_config.yaml         # 集中配置
  build_subject_lexicon.py
  parse_exam_pdf.py
  tag_questions.py
  render_exam_atoms.py
  aggregate_exam_indices.py
  _<subject>_lexicon.json           # 缓存：白名单（按学科生成）
```

**真题词条命名规则**（与现有学科目录的 `01-加法.md` 风格一致但带年份+卷别前缀）：
- 文件名：`YYYY-<卷别简称>-NN.md`，例如 `2024-新课标Ⅱ-08.md`、`2017-理-15.md`
- 卷别简称枚举：`理` / `文` / `新课标Ⅱ` / `全国乙` / `全国Ⅱ` / `新课标`（具体用 `exam_pipeline_config.yaml` 维护映射）
- 2008-2022 年文/理分卷用 `<年份>-<文|理>-NN.md`；2023+ 不分文理用 `<年份>-<卷别>-NN.md`

## 3. 真题词条 frontmatter schema

```yaml
---
title: 2024-新课标Ⅱ-08
aliases: [2024吉林数学新课标Ⅱ-08, 2024年新课标Ⅱ卷数学第8题]
学科: 数学
学段: 高考
省份: 吉林
年份: 2024
卷别: 新课标Ⅱ              # 见上方枚举
文理: 不分                  # 文/理/不分
题号: 8
题型: 选择                  # 选择/填空/解答
分值: 5
考点: [函数, 导数应用, 单调性]    # 必须全部命中现有词条 bare-name
难度: 中                    # 易/中/难（启发式：选择前6=易, 解答倒2=难, 其他=中）
录入状态: 已入库            # 已入库 / 待人工核对（OCR 失真等）；与现有词条 frontmatter `状态` 字段区分，避免被 check_missing.py 误判
来源PDF: ../../素材/真题/吉林/2008-2024·（吉林）数学高考真题/2024年高考数学试卷（新课标Ⅱ卷）（解析卷）.pdf
---

## 题面
（OCR/解析卷提取的题目原文）

## 标准解答
（解析卷答案，简化复述）

## 关联知识点
- [[函数]]
- [[导数应用]]
- [[单调性]]
```

## 4. 反链回填到现有词条

每个被命中的现有词条（如 `数学/265-函数.md`）末尾追加：
```markdown
## 高考真题命中
- [[2024-新课标Ⅱ-08]]
- [[2023-新课标Ⅱ-11]]
（按年份倒序，最多 5 道）
```

由 `aggregate_exam_indices.py` 自动维护这一段（用清晰的 HTML 注释定界 `<!-- exam-backlinks-start -->` / `<!-- exam-backlinks-end -->`，重跑可幂等覆盖该段）。

## 5. 处理流水线

```
PDF 解析卷
  ↓ ① markitdown skill → Markdown 全文
卷子全文 .md（中间产物，docs/superpowers/working/，不入 git）
  ↓ ② parse_exam_pdf.py 按题号正则切分
~22 个 Q&A 块（中间产物 .json）
  ↓ ③ tag_questions.py：白名单候选 + LLM 终审
+ 考点 tag 与缺口词条候选
  ↓ ④ render_exam_atoms.py
真题/<省份>-<学科>/YYYY-*-NN.md
  ↓ ⑤ aggregate_exam_indices.py
反链回填 + 4 份索引
```

### 5.1 第 ③ 步打标策略（核心）

**两阶段**：
- 阶段 A（白名单候选）：从现有 `数学/*.md` 提取 `aliases` + bare-name 作为白名单（约 1500 词），扫描题面+解答，列出所有命中词。
- 阶段 B（LLM 终审）：sonnet 子代理从候选集中选 2-4 个最贴合的考点；同时输出"题面里出现但白名单没有的概念"作为**缺口候选**。

**强约束**：
- LLM 输出必须为合法 JSON（schema 在 prompt 里固定）
- 考点 tag 必须 100% 来自白名单，否则该题 `状态: 待人工核对`
- 每题 tag 数 ∈ [2, 4]
- **打 tag 阶段**用全量白名单（保留所有命中），保证反链完整；**统计阶段**才按 `tag_pool_filters` 过滤（只统计高中数学序号 265-514 + 初中关键概念），避免"加法"等小学词在高频榜单污染。两阶段分离，避免反链丢失

### 5.2 子代理切分

- 每张卷 ≈ 22 题打标 → 1 个 sonnet 子代理一次跑完
- 19 套卷 → 19 个独立子代理任务，可并行 4-6 个
- 主会话只做：脚本编排、词条结构验证、跨卷统计聚合、反链回填

## 6. 脚本可复用性

### 6.1 参数化清单

| 脚本 | 关键参数 | 跨学科/省份切换方式 |
|---|---|---|
| `build_subject_lexicon.py` | `--subject 数学` | 改 `--subject 英语` 即重生成 |
| `parse_exam_pdf.py` | `--pdf <路径>` | PDF 路径独立，与省份学科无关 |
| `tag_questions.py` | `--lexicon _<subject>_lexicon.json` | 词典传参，逻辑通用 |
| `render_exam_atoms.py` | `--province 吉林 --subject 数学 --year 2024 --paper 新课标Ⅱ` | 输出路径与文件名按参数拼接 |
| `aggregate_exam_indices.py` | `--province 吉林 --subject 数学` | 索引文件命名按 `<省份><学科>-*.md` |

### 6.2 `exam_pipeline_config.yaml` 集中维护

```yaml
provinces: [吉林, 北京, 黑龙江]
subjects: [数学, 语文, 英语, 物理, 化学, 生物, 历史, 政治, 地理]

# 卷别归一化映射（PDF 文件名 → 短简称）
paper_aliases:
  "新课标Ⅱ卷": 新课标Ⅱ
  "全国乙卷": 全国乙
  "全国卷Ⅱ": 全国Ⅱ
  # ...

# 题型识别正则（每学科可独立配置）
question_patterns:
  数学:
    选择: '^\d+\.\s*.+\([A-D]\)'
    填空: '^\d+\.\s*.+_+'
    解答: '^\d+\.\s*\(\d+分\)'

# tag 池过滤：仅参与统计的词条范围
tag_pool_filters:
  数学:
    序号范围: [265, 514]      # 高中数学
    额外纳入: [函数, 不等式, 三角函数, 数列, 向量, 导数, 概率, 统计]  # 初中关键概念
```

## 7. 阶段拆分与里程碑

| 阶段 | 工作 | 验证标准 | 工作量 |
|---|---|---|---|
| **P0 脚手架** | 5 个脚本 + 配置 + 白名单生成 + 词条目录建立 | `build_subject_lexicon.py --subject 数学` 跑通，导出 `_math_lexicon.json`（≥ 1500 词） | ~半天 |
| **P1 Pilot** | 跑通 2024 新课标Ⅱ 1 张卷端到端 | 22 题全部入库 + tag 命中 ≥ 90% + 反链写入 ≥ 5 个高频词条 | ~1 小时 |
| **P2 扩量近 5 年** | 2020-2024 共 9 套卷 | 19 套卷的 ~200 题全部入库；高频考点表 v1 | ~半天（4-6 子代理并行） |
| **P3 扩量到 10 年** | 2015-2019 再 10 套卷 | 共 ~420 题入库；4 份索引终稿；反链回填全量完成；缺口词条清单 v1 | ~半天 |

### 7.1 P1 Pilot 验收清单

只有全部通过才进 P2：
- ✅ 22 个 .md 词条 frontmatter 字段齐全且通过 yaml lint
- ✅ tag 全部能在 `_math_lexicon.json` 命中（白名单约束）
- ✅ 至少 1 个考点的反链回填能在 Obsidian 里点击跳转
- ✅ 缺口词条清单非空（缺口检测在工作）
- ✅ OCR/公式失真题数 ≤ 2 道

## 8. 风险与回退

| 风险 | 触发条件 | 回退策略 |
|---|---|---|
| PDF 公式 OCR 失真严重 | P1 时 ≥ 5 道题题面失真 | 暂停流水线，`parse_exam_pdf.py` 接入 `pdfplumber`/`mathpix` 或人工补题面 |
| LLM 打 tag 漂移 | tag 落在白名单外 | 强制 schema + 白名单约束 + 每题 tag 数 ∈ [2,4]；漂移题 → `状态: 待人工核对` |
| 高频考点池被小学词污染 | "加法"等小学词条出现频次异常高 | 统计阶段按 `tag_pool_filters` 过滤（见 6.2），不影响反链完整性 |
| 反链幂等失败 | 二次跑覆盖了非真题区域 | 用 `<!-- exam-backlinks-start/end -->` 注释定界，只覆写区段内 |
