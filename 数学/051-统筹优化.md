# 统筹优化

> **一句话**：把要做的事情合理安排顺序，让等待时间都"干点别的"，从而用最短的时间完成所有任务。
> **English**: Optimization means arranging tasks smartly so waiting time is never wasted — finish everything in the shortest possible time.

---

## 🧒 给 3-6 岁（共读版）

妈妈要给你做早饭：煮蛋要 8 分钟，烤面包要 3 分钟，洗苹果要 2 分钟。

**笨笨的做法：** 先煮蛋，等 8 分钟；再烤面包，等 3 分钟；再洗苹果，等 2 分钟——总共等 **13 分钟**。

**聪明的做法：** 把蛋放进去开始煮，**等蛋的时候**先烤面包，烤完面包**继续等蛋的时候**再洗苹果——蛋煮好了，其他也全做完了！只用了 **8 分钟**！

这就是**统筹优化**——让等待时间不白白浪费！

**共读小活动：** 下次和爸爸妈妈一起准备早餐，讨论一下每件事要多久，看看能不能找到"等这个的时候顺便做那个"的好方法。

## 📚 给 6-12 岁（自读版）

### 经典问题一：沏茶问题

**情景：** 妈妈要给客人沏茶。各项任务时间如下：

| 任务 | 时间 |
|------|------|
| 烧水 | 8 分钟 |
| 洗茶壶 | 1 分钟 |
| 洗茶杯 | 2 分钟 |
| 拿茶叶 | 1 分钟 |

**笨办法（串行）：** 8 + 1 + 2 + 1 = **12 分钟**

**聪明办法（并行）：**
```
[烧水: 8 分钟]
      ↑ 等水开的时候做：
[洗茶壶: 1分] → [洗茶杯: 2分] → [拿茶叶: 1分]  ← 共 4 分钟，烧水还没好
等水一开，马上泡茶！
```
总时间 = **8 分钟**（洗茶壶等可以在等水的时候完成）

**关键思想：** 找出最耗时的那个任务（这里是"烧水"），在等它的时候把其他短任务全塞进去。

### 经典问题二：烙饼问题

**情景：** 锅里只能放 **2 张饼**，每张饼有正反两面，烙一面需要 **1 分钟**。现在要烙 **3 张饼**，最少需要几分钟？

**错误思路（很多人的直觉）：**
- 先烙饼 A（正+反）：2 分钟
- 再烙饼 B（正+反）：2 分钟
- 再烙饼 C（正+反）：2 分钟
- 合计：**6 分钟**

**聪明方法：**

| 第几分钟 | 锅里的情况 |
|----------|-----------|
| 第 1 分钟 | 饼A正面 + 饼B正面 |
| 第 2 分钟 | 饼A反面 + 饼C正面 |
| 第 3 分钟 | 饼B反面 + 饼C反面 |

3 分钟全部搞定！锅始终满载，没有空位浪费。

**公式规律：** n 张饼（n 为奇数时）最少需要 n 分钟（每分钟烙 2 面，共需 2n 面 ÷ 2 = n 分钟）。

### 统筹优化的核心思想

1. **列出所有任务** 及其所需时间
2. **找到关键任务**（最长的那条"主线"）
3. **把空闲时间填满**（用次要任务占据等待时间）
4. **不能缩短关键任务，但能消除等待浪费**

这个思想在现实中无处不在：
- 厨师做多道菜时的排菜顺序
- 工厂生产线的节拍安排
- 你写作业时先写耗脑力的还是先写简单的

## 🎓 给 12+（进阶版）

### 关键路径法（CPM）

在工程管理中，统筹优化发展为**关键路径法（Critical Path Method, CPM）**。

**核心概念：**
- 将所有任务画成**有向网络图**，每条边代表一个任务
- **关键路径**：从开始到结束最长的那条路径
- 关键路径上的任何任务延误都会**直接延误总工期**
- 非关键路径上的任务有"**浮动时间**"（slack time），可以适当延迟

**沏茶问题的 CPM 表示：**
```
开始 → [烧水: 8min] → 结束
开始 → [洗茶壶: 1min] → [洗茶杯: 2min] → [拿茶叶: 1min] → 结束
```
关键路径：烧水（8 分钟），非关键路径总长 4 分钟（有 4 分钟浮动）。

### 线性规划初步

烙饼问题可以用**线性规划**建模。设每分钟锅的使用率为目标，约束条件为：
- 每张饼正面和反面各需烙 1 次
- 锅同时最多容纳 2 张饼

当问题规模很大（比如 100 张饼，锅容量 5 张），就需要用**单纯形法（Simplex Method）**求解。

### 调度问题的复杂度

一般的任务调度问题（m 台机器，n 个任务）是 NP-hard 问题——没有已知的多项式时间算法能求出精确最优解。实际工程中使用：
- **贪心算法**：最短处理时间优先（SPT）
- **启发式算法**：遗传算法、模拟退火
- **整数规划**：精确求解小规模实例

Python 求解简单调度问题：

```python
from itertools import permutations

def min_makespan(tasks, machines=2):
    """
    暴力枚举所有任务顺序，找到最短完成时间
    tasks: 任务列表 [(任务名, 时间), ...]
    """
    best_time = float('inf')
    best_order = None
    for perm in permutations(tasks):
        # 模拟两台机器并行处理
        machine_time = [0] * machines
        for task_name, duration in perm:
            # 分配给最早空闲的机器
            m = machine_time.index(min(machine_time))
            machine_time[m] += duration
        total = max(machine_time)
        if total < best_time:
            best_time = total
            best_order = perm
    return best_time, best_order

# 烙饼问题：3 张饼，每面 1 分钟，锅容量 2
# 等价为：6 个任务(每面1次)，2 台"槽位"并行
tasks = [('A正', 1), ('A反', 1), ('B正', 1),
         ('B反', 1), ('C正', 1), ('C反', 1)]
print(min_makespan(tasks, machines=2))  # 期望: (3, ...)
```

## 🗣️ 家长讲解话术

- 孩子可能问 **"为什么烙 3 张饼不需要 6 分钟？"** → 可以说："因为锅里可以同时放 2 张饼，只要把三张饼聪明地'穿插'着烙，每一分钟锅里都有两张在烤，就不浪费时间了。就像两列火车道，你不会让一条道一直空着吧？"

- 孩子可能问 **"统筹和优化是一个意思吗？"** → 可以说："差不多！统筹是中文说法，意思是'把所有事情通盘考虑'；优化是说'找到最好的方案'。统筹是手段，优化是目标。"

- 孩子可能问 **"我写作业也能用这个方法吗？"** → 可以说："当然！比如发下来一张英语卷子要涂色、一篇语文要写字、数学题要计算。涂色不太费脑子，可以在脑子累的时候做；先写费脑子的数学，再做轻松的涂色——这就是'统筹安排'！"

- **共读小活动：** 列出明天早上起床要做的所有事（刷牙、洗脸、穿衣服、吃饭、收书包），每件事需要几分钟，然后讨论：有哪些事可以"同时做"，排出最快完成的顺序。
