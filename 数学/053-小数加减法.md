# 小数加减法

> **一句话**：小数加减法和整数一样，只要记住**小数点对齐**（不是末位对齐！），其他步骤完全相同。
> **English**: Decimal addition and subtraction work just like whole numbers — the only rule to remember is to line up the decimal points (not the last digits!).

---

## 🧒 给 3-6 岁（共读版）

你去超市买了一块巧克力，花了 **3.5 元**；又买了一包饼干，花了 **2.8 元**。一共花了多少钱？

3.5 + 2.8 = ?

小数里有个小点点——**小数点**，它把整数部分和小数部分分开。

计算的时候，最重要的一件事是：把两个数的**小数点对齐**，就像两个人站队，腰带的位置对齐，才能比谁高谁矮。

**共读小活动：** 下次去超市，让孩子找两件东西的价格标签，试着说出"这个多少元几角，那个多少元几角，合起来要多少钱？"

## 📚 给 6-12 岁（自读版）

### 核心规则：小数点对齐！

**错误做法（末位对齐）：**
```
  3.5
+ 2.80  ← 末位对齐了，但小数点不齐！
------
  5.13  ← 完全错误
```

**正确做法（小数点对齐）：**
```
  3.50   ← 3.5 末尾补 0（不改变大小）
+ 2.80
------
  6.30   ← 小数点也写上！
```

**为什么要小数点对齐？**

因为小数点的位置决定了每一位数字的"权重"：
- 小数点左边第一位 = 个位（1）
- 小数点右边第一位 = 十分位（0.1）
- 小数点右边第二位 = 百分位（0.01）

相同权重的数才能相加减，所以必须让小数点对齐，保证"十分位加十分位，百分位加百分位"。

### 竖式计算步骤

**例题 1：** 计算 5.48 + 3.7

```
步骤 1：小数点对齐，位数不足的末尾补 0
  5.48
+ 3.70

步骤 2：从最低位（百分位）开始，逐位相加
  5.48
+ 3.70
------
  9.18
```

验证：5 + 3 = 8（个位），0.4 + 0.7 = 1.1（进位），0.08 + 0 = 0.08，合计 9.18 ✓

**例题 2：** 计算 8.3 - 4.85

```
步骤 1：小数点对齐，末尾补 0
  8.30
- 4.85

步骤 2：不够减时向前一位借位（和整数减法一样）
  8.30
- 4.85
------
  3.45
```

验证：百分位 0 - 5 不够，借位变 10 - 5 = 5；十分位 2 - 8 不够（借过来少 1），12 - 8 = 4；个位 7 - 4 = 3（借过来少 1）→ 3.45 ✓

### 末尾 0 的处理

计算结果中末尾的 0 可以去掉，数值不变：
- 6.30 = **6.3**（末尾 0 去掉）
- 5.10 = **5.1**
- 3.00 = **3**（变成整数）

**注意：** 中间的 0 不能去掉！6.03 ≠ 6.3

### 实际场景练习

**超市购物：**

| 商品 | 价格 |
|------|------|
| 牛奶 | 4.50 元 |
| 面包 | 3.80 元 |
| 苹果 | 6.75 元 |

总价：4.50 + 3.80 + 6.75 = ?
```
  4.50
  3.80
+ 6.75
------
 15.05 元
```

付了 20 元，找回：20.00 - 15.05 = 4.95 元

## 🎓 给 12+（进阶版）

### 浮点数与计算机中的小数运算

计算机**无法精确表示大多数小数**——因为计算机用二进制，而绝大多数十进制小数不能被二进制有限位数精确表示。

**经典例子：**

```python
>>> 0.1 + 0.2
0.30000000000000004   # 不是 0.3！
```

原因：0.1 在二进制中是无限循环小数（类似 1/3 = 0.33333…），存储时必须截断，产生**舍入误差**。

**IEEE 754 浮点数标准：**

现代计算机用 **IEEE 754** 标准存储小数：
- 单精度（float，32 位）：约 7 位有效数字
- 双精度（double，64 位）：约 15-16 位有效数字

格式：`±  尾数  × 2^指数`（类比科学记数法，但底数是 2）

**如何处理精度问题？**

```python
# 错误做法：直接比较浮点数
0.1 + 0.2 == 0.3   # False！

# 正确做法 1：用 round() 控制精度
round(0.1 + 0.2, 1) == 0.3   # True

# 正确做法 2：用 decimal 模块（精确小数运算）
from decimal import Decimal
Decimal('0.1') + Decimal('0.2') == Decimal('0.3')   # True

# 正确做法 3：金融计算统一用整数（单位：分）
price_milk = 450    # 4.50 元 = 450 分
price_bread = 380   # 3.80 元 = 380 分
total = price_milk + price_bread  # 830 分 = 8.30 元
```

### 小数在不同进制下的行为

| 小数 | 十进制 | 二进制 | 能否精确表示（二进制） |
|------|--------|--------|----------------------|
| 0.5 | 0.5 | 0.1 | ✅ 可以（= 1/2） |
| 0.25 | 0.25 | 0.01 | ✅ 可以（= 1/4） |
| 0.1 | 0.1 | 0.000110011… | ❌ 无限循环 |
| 0.3 | 0.3 | 0.010011001… | ❌ 无限循环 |

规律：只有分母是 2 的幂次的分数（1/2, 1/4, 1/8…）才能用二进制有限位数精确表示。

## 🗣️ 家长讲解话术

- 孩子可能问 **"为什么要小数点对齐，不是末位对齐？"** → 可以说："因为小数点是个'参照系'——它左边是个位、十位……它右边是十分之一、百分之一……。相同单位的才能加减：角加角、分加分。如果末位对齐，就可能把'角'和'分'加在一起，就错了。"

- 孩子可能问 **"3.5 末尾补个 0 变成 3.50，大小会变吗？"** → 可以说："不会！3.5 就是 3 又 5 个十分之一；3.50 是 3 又 50 个百分之一，50 个百分之一 = 5 个十分之一，一样大。末尾的 0 只是占位，不改变数值。"

- 孩子可能问 **"计算机算小数会出错吗？"** → 可以说："会！计算机用二进制，0.1 在二进制里是无限循环的，存不完整。所以银行和超市的程序，会把钱的金额全部乘以 100，全用整数（'分'）来计算，算完再除以 100，这样就不会出错了。"

- **共读小活动：** 用超市收据练习——找到几件商品的价格，用竖式计算总价，再和收据上的合计对比，看算对了没有。
