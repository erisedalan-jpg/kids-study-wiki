# 程序框图

> **一句话**：程序框图是用标准化的图形符号、流程线和文字说明来描述算法每一步骤的执行顺序与逻辑关系的图形工具。  
> **English**: A flowchart is a diagrammatic representation of an algorithm, using standardized symbols and arrows to indicate the sequence and logic of operations.

## 🎓 考点精讲

**定义/原理**  
程序框图的核心是将算法的三种基本逻辑结构——**顺序结构、条件结构、循环结构**——通过图形化方式表达。每个框代表一种操作：起止框（椭圆）表示算法的开始或结束；输入/输出框（平行四边形）表示数据的输入或输出；处理框（矩形）表示赋值或计算；判断框（菱形）表示条件判断，它决定程序的流向。这些框之间用带箭头的流程线连接，构成完整的算法路径。

在高考中，程序框图的考查形式是：给出一段带有已知或未知结构的框图，要求读出其功能或输出结果，也可能要求补全框图内的条件或操作。其背后通常结合**数列求和、累乘、求最大（小）值、辗转相除法、更相减损术、二分法求方程近似解**等具体模型。

**关键方法**  
解答此类问题的标准方法为“**列表追踪法**”：
1. 确定初始值：明确各变量（如 `S`, `k`, `n`）在进入循环前或程序开始时的值。
2. 按步骤执行：从起点开始，按照流程线逐框运行。遇到判断框时，计算条件表达式，若为真则沿“是”分支走，否则沿“否”分支走。
3. 循环结构处理：当进入循环时，在草稿纸上列出每一次循环前后各变量的值，直到满足跳出循环的条件为止。特别注意**循环结束条件的边界**：是“>”还是“≥”，是“当型”循环（先判断后执行）还是“直到型”循环（先执行后判断）。
4. 对照输出：程序运行结束后，查看输出框所指示的变量，结合追踪表得出最终结果。

**典型应用**  
- **累加/累乘型**：例如框图给出 `S=0, i=1`，循环体 `S=S+i, i=i+1`，判断条件为 `i>100?` 则输出 `S`，该框图功能为计算 `1+2+...+100`。这里若误将条件写成 `i≥100` 或 `S=S+i` 与 `i=i+1` 顺序颠倒，结果便不同。
- **条件筛选型**：如给定框图判断一个正整数 `n` 是否为质数。核心是循环变量 `i` 从 2 递增到 `n-1`，若 `n` 能被 `i` 整除，则输出“不是质数”并结束；若循环正常结束，说明没有任何整除，则输出“是质数”。需注意 `n=2` 时的边界情况和整除判断 `n mod i = 0` 的实现。
- **迭代求值型**：如用辗转相除法求两个正整数的最大公约数，框图会反复执行“较大数替换为两数之差或余数”，直到两数相等为止。

## ⚠️ 高频易错点

- **循环终止条件搞反**：当型循环的条件如 `i≤100` 与直到型循环的条件 `i>100` 易混淆，导致循环次数多一次或少一次。  
- **循环体内操作顺序错**：例如在累加 `S=S+i` 之前先执行 `i=i+1`，会使累加缺失首项，或累乘时初值误设为 `0`。  
- **初始值遗漏或错误**：累加变量通常初值为 `0`，累乘变量初值为 `1`，若误设会导致结果错误。对于求最大值的问题，初始值常设为第一个数或一个极值。  
- **分支结构条件遗漏**：判断框的条件表达式书写模糊，例如将“`x<5` 且 `x>1`”简化为 `1<x<5`（框图通常不支持连续不等式，需拆分为逻辑与）。  
- **输出变量混淆**：部分框图执行结束后输出的是循环控制变量而非累加和，学生常忽视输出框的具体变量名。
