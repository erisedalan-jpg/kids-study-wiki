# 算法与框图

> **一句话**：算法是解决某一类问题的明确且有限的规则序列，程序框图（流程图）是用规定的图形符号与流程线直观表示算法步骤的一种图形化工具。
> **English**: An algorithm is a finite sequence of well-defined rules for solving a class of problems, and a program flowchart is a graphical tool that uses standardized symbols and flow lines to visually represent the steps of an algorithm.

## 🎓 考点精讲

**定义/原理**
算法具备五大特征：有穷性（执行有限步后终止）、确定性（每一步含义唯一）、可行性（每一步可通过基本运算实现）、输入（有零个或多个输入）、输出（至少有一个输出）。程序框图由起止框、输入/输出框、处理框、判断框（菱形）、流程线等构成，对应三种基本逻辑结构：

- **顺序结构**：按先后次序依次执行。
- **条件结构**：根据条件 $P$ 是否成立，选择执行 $A$ 框或 $B$ 框（可缺省一路）。
- **循环结构**：重复执行循环体，分为当型（While 型，条件满足时执行循环体）与直到型（Until 型，执行循环体直到条件满足）。循环结构必须包含终止条件，否则构成死循环。

**关键方法**
解决框图问题的核心是 **循框执行**（手算跟踪）：从“开始”进入，沿流程线方向依次模拟每个框的操作，遇到判断框严格按条件分流，遇到循环结构逐轮记录变量变化，直至循环结束或到达“结束”。求解结果输出值是高频考法；补全判断框中的条件则需要逆推终端值与中间变量的关系。

**典型应用**
高考通常以“根据框图写出运行结果”或“补全框图内条件/处理语句”两种形式考查，常结合数列（累加/累乘）、函数分段求值、求最大/最小值等真实数学情境。例如：给定一个计算 $S=1^2+2^2+\dots+n^2$ 的框图，其中循环体为 $S=S+k^2$ 且 $k=k+1$，要求写出 $n=5$ 时的输出。

## ⚠️ 高频易错点

- **循环终止条件的混淆**：当型循环（先判断后执行）与直到型循环（先执行后判断）的判断框出口逻辑相反，学生易因混淆循环类型而多算或少算一轮。
- **变量初始值与循环次数计算错误**：当循环变量 $i$ 初始为 $0$ 而框图中执行 $i=i+1$，循环体执行次数常被错误等同于 $i$ 的终值，实际上需要检查最后一次执行后是否因不满足条件退出。
- **判断框条件边界值偏差**：补全条件时，极易在“$>$”与“$\ge$”、“$<$”与“$\le$”之间选错，尤其是在输出最后一个满足条件的值时。
- **输出结果对应关系错位**：框图执行过程中某个变量被重复赋值，最终输出的是该变量的最新值而非历史累加值，学生可能误取循环中间快照作为答案。
