# 赋值运算

> **一句话**：在算法中将一个确定的值或表达式的计算结果存储到指定变量中的基本操作，用符号“**=**”或“**←**”表示。
> **English**: The operation of storing a value or the result of an expression into a specified variable, typically denoted by "**=**" or "**←**".

## 🎓 考点精讲

**定义/原理**：在数学算法（特别是程序框图与算法语句）中，赋值运算 `A = b`（或 `A ← b`） 的含义是：先计算右边表达式 `b` 的值，然后将该值赋给左边的变量 `A`。其核心是**变量值的覆盖与更新**，而非代数恒等关系。例如，执行 `x = x + 1` 在数学方程中没有解，但在算法中表示将变量 `x` 的当前值加上 1 后再存回 `x` 中。

**关键方法**：处理赋值运算问题需严格遵循“**先右后左**”的执行顺序，并跟踪变量值的**动态变化**。标准步骤如下：
1.  **查表法**：列出所有变量，按算法步骤逐行执行。
2.  **状态更新**：每当执行到赋值语句，立即用表达式当前计算出的新值**覆盖**变量原有的旧值。
3.  **循环初始化**：在进入循环结构前，为累加变量（如 `S = 0`）或累乘变量（如 `T = 1`）赋初始值。

**典型应用**：高考主要在**程序框图题**和**算法语句补全题**中考查。
-   **场景一（累加求和）**：设计算法计算 $1+2+3+\dots+100$。需设定累加变量 `S` 初值为 0，计数变量 `i` 初值为 1，循环体执行 `S = S + i`，`i = i + 1`。
-   **场景二（变量交换）**：交换变量 `a` 和 `b` 的值，需引入中间变量 `t`，执行指令序列：`t = a`，`a = b`，`b = t`。其中任何一步顺序错误都会导致数据丢失。

## ⚠️ 高频易错点

-   **混淆赋值与等于**：将 `x = y`（表示将 `y` 的值赋给 `x`）误解为判断 `x` 与 `y` 是否相等。在算法框图中，判断相等使用 `x == y` 或菱形框内的条件表达式。
-   **变量未初始化**：在使用累加变量前忘记赋初值 `S = 0`，导致输出结果为一个随机或未定义的值（尽管高考题中通常给定初值，但理解原理时易忽略）。
-   **执行顺序错乱**：忽略“先右后左”。例如 `A = B` 和 `B = A` 效果截然不同；或在连续赋值 `m = n = 1` 中，错误理解赋值从右向左进行的传递性（先将 `1` 赋给 `n`，再将 `n` 的值赋给 `m`）。
-   **循环边界与终值**：在循环结构中，未准确记录最后一次赋值后变量的变化，导致输出结果比正确值多一次循环或少一次循环（即“边界差一错误”）。
