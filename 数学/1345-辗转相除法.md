# 辗转相除法

> **一句话**：辗转相除法是求两个正整数最大公约数的高效算法，核心原理为 gcd(a, b) = gcd(b, a mod b)，反复执行直到余数为零。
> **English**: The Euclidean algorithm is a method for computing the greatest common divisor of two integers, based on the principle that gcd(a, b) = gcd(b, a mod b) until the remainder becomes zero.

## 🎓 考点精讲

- **定义/原理**：设 a, b∈ℕ⁺，不妨 a ≥ b。由带余除法 a = bq + r (0 ≤ r < b)，则 gcd(a, b) = gcd(b, r)。重复此过程：将 (a, b) 替换为 (b, r)，直到余数 r = 0，此时的除数 b 即为最大公约数。该性质基于：若 d 整除 a 与 b，则 d 整除 a − bq = r；反之亦然，故公因数集合不变。
- **关键方法**：标准化步骤：① 输入 a, b；② 计算 r = a mod b；③ 若 r = 0，输出 b 结束；④ 否则令 a ← b，b ← r，返回②。算法必在有限步内终止，且可推广至多项式求公因式。
- **典型应用**：高考常见设问：给定两个正整数（如 8251 与 6105），直接计算其最大公约数；在算法与程序框图中补全判断框或处理框中的条件；利用 gcd 求最小公倍数 lcm(a, b) = a·b / gcd(a, b)；判断两数是否互质（gcd = 1）；结合线性丢番图方程 ax + by = c 的整数解存在条件（c 能被 gcd(a, b) 整除）。题型以选择题、填空题或流程填空题为主。

## ⚠️ 高频易错点

- **除数与被除数顺序混乱**：学生可能错误固定用大数除小数，但在迭代中未正确交换变量，导致公式变成 gcd(a, b) = gcd(a mod b, b) 而出错。确保严格执行 gcd(原除数, 余数) 的顺序。
- **终止条件误判**：误以为余数为 1 或某一较小值时即可停止，实际上必须余数为 0 时，当前的除数才是最大公约数。若提前终止，会得到错误结果。
- **求最小公倍数时忘记约分**：直接用两数乘积作为 lcm，未除以最大公约数，这是高频错误。正确公式为 lcm(a, b) = |a·b| / gcd(a, b)。
- **多项式辗转相除时忽略系数**：在多项式版本中，允许乘以非零常数，但学生容易直接套用整数方法而忽视系数处理，导致结果差一个常数倍（待家长核对）。
