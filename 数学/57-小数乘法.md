# 小数乘法

> **一句话**：小数相乘，先当整数算，再根据两个因数小数位数之和确定积的小数点位置。
> **English**: To multiply decimals, multiply as if they were whole numbers, then place the decimal point so the product has as many decimal places as the two factors combined.

---

## 🧒 给 3-6 岁（共读版）

你知道 1.5 是什么意思吗？就是"一块半"——1 块钱再加半块钱！

如果买了 2 个 1.5 元的糖，一共要多少钱？

我们可以想：2 个 1 元 = 2 元，2 个半元 = 1 元，合在一起就是 **3 元**。

小数乘法就是这样——把"有点数"的数字乘起来！

**共读小活动：** 拿 3 张 1 元纸币，再把其中半张用手捂住——"1.5 元"是多少？两份 1.5 元合在一起数一数。

## 📚 给 6-12 岁（自读版）

### 小数乘整数：先当整数，再定点

**步骤：**
1. 把小数当作整数来乘（暂时忘掉小数点）
2. 数清楚原来的小数一共有几位小数
3. 在积的右边往左数"同样多"的位数，放上小数点

**例 1：** 1.2 × 3 = ？

```
想成：12 × 3 = 36
1.2 有 1 位小数，所以积也有 1 位小数
36 → 3.6
答：1.2 × 3 = 3.6
```

**生活情境：** 买了 3 包饼干，每包 1.2 元，共花多少钱？答：3.6 元。

---

### 小数乘小数：位数相加定小数点

**核心规则：** 积的小数位数 = 两因数小数位数之**和**。

**例 2：** 1.2 × 0.5 = ？

```
想成：12 × 5 = 60
1.2 有 1 位小数，0.5 有 1 位小数，1+1=2 位
60 → 0.60 → 去掉末尾的 0 → 0.6
答：1.2 × 0.5 = 0.6
```

**例 3：** 2.35 × 1.4 = ？

```
235 × 14 = 3290
2.35 有 2 位小数，1.4 有 1 位小数，2+1=3 位
3290 → 3.290 → 去掉末尾 0 → 3.29
答：2.35 × 1.4 = 3.29
```

**验证：** 2 × 1 = 2，所以答案在 2~3 之间，3.29 合理！估算是保证不出错的好方法。

---

### 末尾 0 要去掉

小数乘法算完后，如果积的**末尾有 0**，必须去掉。

- 0.4 × 0.5 = 0.20 → 应写作 **0.2**
- 2.5 × 4 = 10.0 → 应写作 **10**

为什么？因为 0.20 和 0.2 大小一样，末尾的 0 多余，简洁为好。

---

### 积的近似值（四舍五入）

有时不需要精确答案，可以用**四舍五入**取近似值。

**例 4：** 3.14 × 2.7 ≈ ？（保留 1 位小数）

```
精确答案：3.14 × 2.7 = 8.478
保留 1 位小数：看第 2 位小数是 7 ≥ 5，进 1
答：≈ 8.5
```

**符号：** ≈ 读作"约等于"，表示近似（English: approximately equal to）。

---

### 竖式写法

```
    2 . 3 5
  ×   1 . 4
  --------
    9  4  0   ← 235 × 4
  2 3  5  0   ← 235 × 1（实为 × 10，写在下一行左移一位）
  --------
  3 2  9  0
```

从 3290 定 3 位小数点 → **3.290 → 3.29**

## 🎓 给 12+（进阶版）

### 为什么"位数相加"是对的？

设 a 有 m 位小数，即 a = A / 10^m（A 为整数）；
设 b 有 n 位小数，即 b = B / 10^n（B 为整数）。

则：a × b = (A × B) / 10^(m+n)

这证明了积的小数位数恰好是 m + n。这不是"规则"，而是十进制分数乘法的必然结果。

---

### 浮点数精度问题（计算机中的小数乘法）

计算机用**浮点数（floating-point number）** 存储小数，遵循 IEEE 754 标准。
大多数小数（如 0.1）在二进制中是**无限循环小数**，计算机只能存储有限位数的近似值。

**经典陷阱：**

```python
>>> 0.1 + 0.2
0.30000000000000004   # 不是 0.3！
>>> 0.1 * 3
0.30000000000000004
```

原因：0.1 在二进制 = 0.0001100110011…（无限循环），截断后累积误差。

**工程解决方案：**
- 金融计算：使用**定点数**或 Python 的 `decimal` 模块
- 精度比较：用 `abs(a - b) < 1e-9` 而非 `a == b`
- 科学计算：了解机器精度 ε ≈ 2.2 × 10⁻¹⁶（双精度浮点）

---

### 乘法与量纲分析

在物理中，小数乘法要带单位：

> 速度 1.5 m/s × 时间 2.4 s = 路程 **3.6 m**

单位也"相乘"：m/s × s = m。量纲分析是检验物理公式正确性的工具，也是初中物理的基础技能。

## 🗣️ 家长讲解话术

- 孩子可能问 **"为什么要数位数放小数点，不能直接对齐？"** → 可以说："小数乘法先把点去掉当整数算，这样更简单。算完之后，乘了多少个 '十分之一'，就要还回来多少位小数。两个因数各有几位，积就一共要退几位。"

- 孩子可能问 **"末尾的 0 为什么要去掉？"** → 可以说："3.20 和 3.2 是同一个数——小数末尾的 0 不改变大小，就像整数前面多个 0 不变大小一样。去掉是为了写法简洁。"

- 孩子可能问 **"计算器算 0.1×3 为什么不是 0.3？"** → 可以说："计算机用二进制存数，0.1 没法精确表示，只能存个近似值。这是计算机的小小缺陷，科学家已经研究出了解决办法，但普通计算器有时会暴露这个问题。"
