# 循环结构

> **一句话**：循环结构是算法框图中通过判断条件控制某一操作（循环体）反复执行的控制结构，是高考算法与程序框图模块的核心考点。
> **English**: A loop structure is a control structure in algorithm flowcharts that repeatedly executes an operation (the loop body) based on a condition, forming a core topic in the algorithms and program flowcharts section of the Chinese Gaokao.

## 🎓 考点精讲

**定义与原理**：在数学学科（算法初步）中，循环结构指从某处开始，按照一定条件反复执行某一处理步骤的结构。反复执行的处理步骤称为**循环体**，控制循环是否继续的条件称为**循环终止条件**。循环结构分为**当型**（while型，先判断后执行）与**直到型**（until型，先执行后判断）两类。其数学本质是描述有限步骤内的迭代过程，如累加求和、累乘求积、数列递推等。

**关键方法**：高考解答此类问题的标准步骤为：
1. **分析循环三要素**：确定循环变量及其初始值、循环体（执行什么运算）、循环终止条件。
2. **逐次迭代执行**：按循环条件，逐次计算循环变量、累加/累乘变量等的值，列出每次循环后的关键变量值。
3. **确定输出结果**：当循环终止条件满足时，输出最终结果；注意区分最后一次循环是否执行。常见技巧是利用数列通项或求和公式直接验证最终结果。

**典型应用**：高考中常以程序框图形式设问，要求输出指定输入下的结果，或补全判断框内的条件。典型场景包括：
- 累加求和：如计算 $S = 1 + 2 + \dots + n$，或 $S = 1 + \frac{1}{2} + \dots + \frac{1}{n}$。
- 累乘求积：如计算 $n!$。
- 数列递推：根据递推关系 $a_{n+1} = f(a_n)$ 迭代求某一项。
- 条件筛选：从若干数据中筛选满足特定条件的项（如统偶数个数）。

## ⚠️ 高频易错点

- **循环终止条件混淆**：未分清当型和直到型的区别——当型在条件不满足时退出循环体（判断“是”才执行），直到型在条件满足时退出循环（执行到条件为“真”时跳出）。
- **循环次数计算偏差**：临界时多算或少算一次循环，尤其在循环变量初值与终止条件边界处（如初值为0、判断 `i ≤ n` 还是 `i < n` 直接导致循环次数不同）。
- **输出变量混淆**：循环体内有多个变量同时迭代变化时，误把循环变量当累加/累乘变量输出，或输出的是循环结束时的变量值而非最终期望值。
- **循环体内运算顺序错误**：框图中“累加”与“循环变量递增”的先后顺序直接影响输出结果，未按流程图箭头逐一执行、仅凭直觉推算极易出错。
