# 算法与程序框图

> **一句话**：算法是解决某一类问题的一系列明确、有限的步骤，程序框图是用标准化的图形符号和流程线来直观描述算法的工具。
> **English**: An algorithm is a finite sequence of well-defined steps for solving a problem, and a program flowchart is a graphical representation of an algorithm using standardized symbols.

## 🎓 考点精讲

**定义/原理**  
算法需满足确定性、有穷性、可行性、输入（可有零个）和输出（至少一个）。程序框图（流程图）由起止框、输入/输出框、处理框、判断框、流程线等组成，核心是三种基本逻辑结构：
- **顺序结构**：按框的顺序依次执行；
- **条件结构**：根据条件判断选择不同的执行路径，常见框图用菱形表示判断，对应数学中的分段函数、分类讨论；
- **循环结构**：重复执行某段操作，分为当型循环（先判断后执行，可能一次都不执行）和直到型循环（先执行一次再判断，至少执行一次）。循环结构常用来求累加和、累乘、迭代等。

用公式表述：在循环求和的程序中，若循环体为 `S = S + a_i`，则最终 `S = Σa_i`，循环变量 `i` 的初始值、步长与终止条件决定了求和的范围。

**关键方法**  
高考中此类问题主要考查阅读程序框图并得出结果，或根据功能补全判断框/处理框。解题关键步骤：
1. 识别框图包含的结构（顺序、条件、循环）；
2. 从起始框开始，按流程线箭头方向逐步模拟，明确每一次循环中变量的值；
3. 注意循环终止条件，特别是指向判断框的“是/否”出口所对应的含义（满足条件时执行循环体还是跳出循环）；
4. 将程序框图与数学问题对应，例如二分法求函数零点、求最大公约数（辗转相除法）、数列求和或求项。

**典型应用**  
高考常见设问：给定程序框图，求输出的数值。例如，框图实现的是计算 `1+2+3+…+100` 的和，框图中出现循环变量 `i` 和累加变量 `S`，初始 `S=0, i=1`，循环条件 `i≤100`，循环体 `S=S+i, i=i+1`，则最终输出 `S=5050`。又如利用条件结构给出分段函数的求值过程，根据输入的自变量值判断所在区间，再代入对应的表达式计算输出。

## ⚠️ 高频易错点

- **累加/累乘变量初始化错误**：在进入循环前未将存放累加和的变量置0、存放累乘积的变量置1，导致结果出错。
- **循环条件与出口逻辑混淆**：未看清“是满足条件循环”还是“是满足条件退出”，导致循环次数多一次或少一次。例如将当型循环误认为直到型循环。
- **循环变量更新顺序与终止值判断错误**：在循环体内变量的更新顺序不当，或对循环终止值的临界情况判断不清，如要求求和至100，却写成了 `i<100`，漏掉了最后的项。
- **混淆顺序结构与选择结构的嵌套**：尤其在处理较复杂数学问题时，将需要同时判断的条件全部并列，导致逻辑分支缺失或重复计算。
